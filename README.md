# 🧾 Invoice Manager

Локальное Java-приложение (Spring Boot) для удобного управления контрагентами, автоматического формирования счетов и их экспорта в формат PDF с автоматической подстановкой факсимиле (печати и подписей) организации.

--

## 🛠 Стек технологий
* **Backend:** Java 21, Spring Boot 3 (Spring Data JPA, Spring Web)
* **Шаблонизатор:** Thymeleaf (генерация HTML-представления счетов)
* **База данных:** PostgreSQL (локальный сервер)
* **Экспорт документов:** Flying Saucer / iText (рендеринг HTML-кода в PDF)

---

## 🗄 Архитектура и работа с данными
* **Автономность:** Приложение спроектировано для работы в виде независимого исполняемого `.jar` файла на любой ОС.
* **Хранение медиафайлов:** В отличие от стандартных веб-приложений, печать организации и подписи должностных лиц хранятся **напрямую в базе данных PostgreSQL** в виде бинарных массивов байт (`byte[]` / тип `BYTEA` в таблице `my_organizations`). Это позволяет переносить базу данных на другой компьютер через обычный дамп без потери картинок и без привязки к путям файловой системы.
* **Рендеринг PDF:** При экспорте счета метод `exportToPdf` извлекает массивы байт из БД, налету кодирует их в Base64-строки и динамически подставляет в HTML-шаблон `invoice-pdf.html` вместо текстовых маркеров `src="org-stamp"`, `src="boss-sig"` и `src="book-sig"`.

## 🚀 Как запустить проект локально

### Шаг 1. Настройка базы данных
1. Установите PostgreSQL и создайте пустую базу данных с именем `invoice_db`.
2. В файле `src/main/resources/application.properties` укажите ваши доступы (по умолчанию настроен пароль `qwe987qwe` для пользователя `postgres`).

### Шаг 2. Сборка исполняемого файла
Выполните сборку проекта через Maven:
```bash
mvn clean package
```
В папке `target` появится файл `invoice-1.0-SNAPSHOT.jar` (или переименованный `im.jar`).

### Шаг 3. Запуск приложения
Перенесите `.jar` файл в любую удобную папку на компьютере и запустите его из Терминала/Командной строки:
```bash
java -jar "invoice manager.jar"
```
После запуска откройте браузер и перейдите по адресу: **`http://localhost:8080`**

## 🔄 Резервное копирование и перенос на другой ПК
Для переноса программы (например, с macOS на Windows) достаточно:
1. Экспортировать бэкап базы данных через **pgAdmin** (нажмите правой кнопкой на `invoice_db` ➡️ **Backup** ➡️ формат *Plain*, файл `backup.sql`).
2. На целевом компьютере установить PostgreSQL, создать пустую `invoice_db` и сделать **Restore** из вашего файла.
3. Перенести файл `.jar` и запустить его.
